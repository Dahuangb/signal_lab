# Signal Lab

> 主播作为一名通信工程的本科生，在学习专业课时，总是深感干瘪的公式与真实的物理世界之间存在着巨大的 Gap。很多时候，理解公式只能靠枯燥的数学推导，极少有直观的、可视化的方法能让我“顿悟”这些抽象概念背后的物理含义。因此，我开发了这个交互式的实验平台，希望能用直观的图形化交互，帮助大家跨越这道鸿沟。

Signal Lab 是一个交互式的信号处理与通信原理可视化实验平台。通过直观的交互界面和实时渲染的波形图，帮助学习者深入理解抽象的信号与系统概念。

## 🌟 核心功能模块

项目目前涵盖了通信电子类专业三大核心基础课的交互式图解与模拟：

### 1. 信号与系统 (Signals and Systems)
* **傅里叶级数与傅里叶变换**：从多个方波谐波的动态合成开始，直观演示时域与频域的映射关系。
* **吉布斯现象 (Gibbs Phenomenon)**：可视化方波合成在间断点处的过冲现象，探讨截断效应。
* **卷积操作 (Convolution)**：通过动画逐步图解线性卷积的“反转、平移、相乘、积分”过程。
* **冲激响应与阶跃响应**：动态展示 LTI 系统对基本测试信号的响应波形。

### 2. 数字信号处理 (Digital Signal Processing - DSP)
* **采样定理与混叠**：实时调整采样率，直观观察时域采样点与频域混叠现象 (Aliasing)。
* **离散傅里叶变换 (DFT) 与 FFT**：展示栅栏效应 (Picket Fence Effect)、频谱泄露以及加窗处理的效果。
* **离散时间系统与卷积**：对比演示循环卷积 (Circular Convolution) 与线性卷积的差异。
* **数字滤波器设计 (Filter Design)**：提供冲激响应与零极点图 (Pole-Zero Plot) 的交互，实时分析系统的幅频响应、相频响应与群延迟。

### 3. 通信原理 (Communication Principles)
* **模拟调制 (AM/FM)**：实时调节载波频率、调制指数等，观察时域波形与频谱变化，深入理解 Carrier Null 等现象的数学本质。
* **数字调制星座图 (Constellation)**：支持 BPSK, QPSK, 16QAM 等调制方式，加入高斯白噪声 (AWGN) 生成点云，直观展示 I/Q 正交信号的物理意义与接收端的判决边界。
* **眼图与码间串扰 (Eye Diagram & ISI)**：
  * **码间串扰 (ISI)**：图解多径效应与带宽受限导致的波形拖尾叠加。
  * **奈奎斯特第一准则**：可视化展示为何在最佳采样点 (t=0) 其他波形恰好经过绝对过零点。
  * **升余弦成型滤波**：对比不同滚降系数 (Roll-off $\alpha$) 下的时域波形拖尾与频带利用率权衡。

## 🚀 本地运行与配置指南

### 1. 环境要求
请确保您的电脑上安装了 Node.js（推荐 v18 或更高版本）和 npm。

### 2. 克隆项目
将本仓库克隆到您的本地：
```bash
git clone https://github.com/Dahuangb/signal_lab.git
cd signal_lab/signal-lab
```

### 3. 安装依赖
在 `signal-lab` 目录下执行以下命令，安装项目所需的全部依赖：
```bash
npm install
```

### 4. 启动开发服务器
执行以下命令启动本地开发环境：
```bash
npm run dev
```
启动成功后，终端会输出一个本地访问地址（通常是 `http://localhost:5173`），在浏览器中打开该地址即可开始体验。

### 5. 构建生产版本
如果您想将项目打包并部署到线上环境，可以运行：
```bash
npm run build
```
打包生成的文件将存放在 `signal-lab/dist` 目录下。

## 📁 目录结构说明

```text
signal-lab/
├── src/
│   ├── components/      # 通用 UI 组件 (导航栏库、参数滑块、Canvas 包装器)
│   ├── engine/          # DSP 数学引擎 (卷积计算、傅里叶变换、噪声生成、调制算法)
│   ├── pages/           # 页面级组件 (各个具体的实验模块页面)
│   ├── renderer/        # Canvas 底层绘图核心库 (网格、坐标轴、高频轨迹绘制)
│   ├── store/           # 全局状态管理 (Zustand)
│   ├── App.tsx          # 路由入口与布局定义
│   └── main.tsx         # React 挂载点
├── public/              # 静态资源 (图标等)
└── package.json         # 项目依赖与脚本配置
```
